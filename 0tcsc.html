Complexity: An algorithm is a well-defined list of steps for solving a particular problem. One major purpose of this topic is to develop efficient algorithms for proccessing of our data. The time and space it uses are two major measures of the efficiency of an algorithm. The complexity of an algorithm is the function which gives the running time and/or space in terms of the input size.
There are two types of complexity:Time Complexity & Space Complexity

When a program is executed, the following steps are being proccessed:

  • Assignment Operation
  • Comparison Operation
  • Mathematical Operation
  • Function calling
  • Proccesses inside Function

Complexity of these steps are also diiferent. Complexity is expressed by "Big-Oh (O) notation. Generally, Time complexity of "Function calling" is ignored during calculation. Time complexity for each Assignment Operation, Comparison Operation and Mathematical Operation is O(1).

Let's take a look on some examples:

Example 01: 

                #include<stdio.h>
                    int main() {
                        int n1, n2, result;
                        
                        n1=20; <---1
                        n2=30; <---2
                        result=n1+n2; <---3&4
                        
                        return 0;
                    }
In this program, we can see there are 3 assignment operations and 1 mathematical operations. So what is the time complexity? O(1+1+1+1)=O(4)..? No, the complexity is O(1). The value of n1 or n2 can be anything. Numbers of operation are constant against the input n. Even if the input is 10000000, that 4 operations remain the same.
