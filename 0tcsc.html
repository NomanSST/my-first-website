Complexity: An algorithm is a well-defined list of steps for solving a particular problem. One major purpose of this topic is to develop efficient algorithms for proccessing of our data. The time and space it uses are two major measures of the efficiency of an algorithm. The complexity of an algorithm is the function which gives the running time and/or space in terms of the input size.
There are two types of complexity:Time Complexity & Space Complexity

When a program is executed, the following steps are being proccessed:

  • Assignment Operation
  • Comparison Operation
  • Mathematical Operation
  • Function calling
  • Proccesses inside Function

Complexity of these steps are also diiferent. Complexity is expressed by "Big-Oh (O) notation.
Time Complexity: Time complexity is defined as the amount of time taken by an algorithm to run, as a function of the length of the input. It measures the time taken to execute each statement of code in an algorithm. Generally, Time complexity of "Function calling" is ignored during calculation. Time complexity for each Assignment Operation, Comparison Operation and Mathematical Operation is O(1).

Let's take a look on some examples:

Example 01: 

    #include<stdio.h>
        int main() {
            int n1, n2, result;
            
            n1=20; <---1
            n2=30; <---2
            result=n1+n2; <---3&4
            
            return 0;
        }
In this program, we can see there are 3 assignment operations and 1 mathematical operations. So what is the time complexity? O(1+1+1+1)=O(4)..? No, the complexity isO(1). The value of n1 or n2 can be anything. Numbers of operation are constant against the input n. Even if the input is 10000000, that 4 operations remain the same.

Now check another code to determine its time complexity.

Example 02: 
    #include<stdio.h>
    int main() {
        int n, result;
        
        scanf("%d",&n);
        result=n*(n+1)/2; <--1,2,3&4
        printf("result=%d\n",result);
        
        return 0;
    }
In this program, there are 3 mathematical operations and 1 assignment operation. The number of operations doesn't depend on the value of n. So, the time complexity is O(1).

Now check this code below and try to understand its time complexity.

Example 03: 
#include<stdio.h>

    int main() {
        int i, n, result=0;
        scanf("%d", &n);
        for(i=0; i<n; i++){<---
            result=result+i;<--1&2
        }
        printf("result=%d\n", result);
        return 0;
    }
In this program, there are 2 operations inside the loop. The loop runs n times according to the input n. If n=10, number of operation=20; for n=100, operations=200. So, for the value of n, the number of operations is 2n. So, the time complexity is O(2*n)=2*O(n)=O(n).

Now let's look at another program:

Example 04: 
#include <stdio.h>
int main() 
{
    int i,j,n,count=0;
    scanf("%d", &n);
    for(i=0; i<n; i++){<---
        for(j=0; j<n; j++){<---
            count = count + 1;<--1&2
        }
    }
    printf("n=%d  count=%d\n", n, count);
    return 0;
}
In the example-04, there are 2 operations inside the nested loop. The loop "i" runs n times and for every "i" turn, loop "j" runs for n times. So the total loop run is n*n=n2. If n=1,loop run=1; n=2, loop run=4; n=3, loop run=9... So, for the input n, the number of operations is also n2. In this case, the time complexity is O(n*n)=O(n2).
